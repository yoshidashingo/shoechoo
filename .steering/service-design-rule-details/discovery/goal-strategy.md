# Goal Setting & Strategy

**Assume the role** of a business strategist / service designer

**Always executes.** サービスデザインの起点として、ビジネスゴールと戦略を明確にする。

## Prerequisites
- サービスアイデアまたはビジネス課題がユーザーから提示されていること

## Execution Steps

### Step 1: ユーザーリクエストの分析

以下の観点でリクエストを分析する:

#### 1.1 サービスの方向性
- **新規サービス**: ゼロからのサービス立ち上げ
- **既存サービス拡張**: 既存サービスへの機能/メニュー追加
- **サービス改善**: 既存サービスの品質・効率向上
- **ピボット**: 既存サービスの方向転換

#### 1.2 対象市場
- **B2B**: 法人向け
- **B2C**: コンシューマー向け
- **B2B2C**: プラットフォーム型

#### 1.3 成熟度
- **探索段階**: アイデアレベル、仮説未検証
- **検証段階**: 一部仮説は検証済み
- **拡大段階**: PMF達成後のスケール

### Step 2: ビジネスゴールの設定

以下をユーザーと整理する:

1. **ミッション**: このサービスが実現する社会的価値
2. **ビジョン**: 3年後の目指す姿
3. **ゴール**: 具体的・計測可能な目標（1年以内）
4. **目的**: なぜこのゴールを達成する必要があるか

### Step 3: Why Now? / Why Us?

#### Why Now?（なぜ今やるのか）
以下を整理する:
- 市場環境の変化（技術、規制、競合、顧客ニーズ）
- タイミングの根拠（2年前でも2年後でもなく今である理由）
- 機会損失のリスク（やらなかった場合に失うもの）

#### Why Us?（なぜこの会社がやるのか）
以下を整理する:
- 自社の強み・アセット（技術力、顧客基盤、ブランド、ノウハウ）
- 競合に対する優位性
- 自社だからこそできること

### Step 4: 中長期計画

| 時期 | プロダクト/サービスの変化 | 市場/社会の変化 |
|------|----------------------|---------------|
| 半年後 | | |
| 1年後 | | |
| 2年後 | | |

### Step 5: 質問ファイルの生成

情報が不足している場合:
1. `aidlc-docs/service-design/discovery/goal-strategy-questions.md` を作成
2. AI-DLCの質問フォーマット（question-format-guide.md）に従う
3. ユーザーの回答を待つ
4. 回答を反映して成果物を作成

### Step 6: 成果物の作成

`aidlc-docs/service-design/discovery/goal-strategy.md` を作成:
- `common/output-templates.md` のテンプレートに従う
- ユーザーの入力と分析結果を統合
