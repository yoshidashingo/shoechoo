# 文書構成とパラグラフ管理

## 1. 文書全体の構成

### ルール

- **RULE-STRUCT-001**: 冒頭に文書の目的を簡潔に記載すること。読者が「この文書は何のためか」を瞬時に判断でき、不要な読者が即座に離脱できるようにする
- **RULE-STRUCT-002**: 結論を先に提示し、その後に根拠・詳細を説明すること。日本語の伝統的な結論後置を避ける
- **RULE-STRUCT-003**: エレベーターピッチ（30秒）で説明できない内容は、論理に欠陥がある可能性を疑うこと
- **RULE-STRUCT-004**: 上位概念から下位概念へ展開する逆ピラミッド構造を採用すること。最初の文が全体を示し、次の文が具体化する階層的な流れにする

## 2. シリーズ記事の構成

### ルール

- **RULE-STRUCT-018**: シリーズ記事は「冒頭にその回のポイントまとめ → 本文 → まとめ・次回予告」の構成を統一すること

## 3. パラグラフ管理

### ルール

- **RULE-STRUCT-005**: 1つのパラグラフには1つの主題のみを含めること
- **RULE-STRUCT-006**: パラグラフの冒頭にトピックセンテンス（そのパラグラフの主題を示す文）を配置すること
- **RULE-STRUCT-007**: パラグラフは視覚的にも意味的にも適切な長さを保つこと。長すぎるパラグラフは分割する
- **RULE-STRUCT-008**: パラグラフ間の論理的なつながりを明示すること。読者が「次に何が来るか」を予測できる流れにする

## 4. 文書の長さ

### ルール

- **RULE-STRUCT-009**: 意思決定や議論を目的とする文書は、15〜20分で読了できる分量を目標とすること
- **RULE-STRUCT-010**: 詳細情報は本文ではなくAppendix（付録）に記載し、本文には重要な情報のみを含めること
- **RULE-STRUCT-011**: 関連情報への参照はハイパーリンクで補完し、本文の肥大化を防ぐこと

## 5. 数値・データの提示

### ルール

- **RULE-STRUCT-012**: 定性的な表現（「速い」「大きい」等）ではなく、具体的な数値（「p99レイテンシ100ms→10ms」等）を用いること
- **RULE-STRUCT-013**: 図表を掲載する場合、自然言語による説明も併記すること。図表なしでも内容が伝わる文章であること
- **RULE-STRUCT-014**: グラフやデータの出典を明示すること

## 6. 箇条書きの使用

### ルール

- **RULE-STRUCT-015**: 箇条書きの濫用を避けること。箇条書きは行間の読みを読者に強制し、読者間で情報のブレが生じやすい
- **RULE-STRUCT-016**: 論理的な説明や因果関係の記述には、箇条書きではなくパラグラフ形式を用いること
- **RULE-STRUCT-017**: 箇条書きは、並列な項目の列挙やチェックリストなど、明確に適切な場面に限定して使用すること
- **RULE-STRUCT-019**: 引用ブロック（`>`）内で箇条書きを使用する場合、引用の冒頭テキストと箇条書きの間に空行（`>`のみの行）を挿入すること。これにより、引用内の箇条書きが正しくMarkdownとしてレンダリングされる
