# Technical Writing Rule Details

## 目的

本ルール群は、AI-DLCプロセスで生成されるすべての技術文書の品質を担保するための執筆基準を定義する。

## 基本原則

> 文書から得られるアウトプットが明確で、読み手のレベルによらず一定に伝わること。

技術文書の品質は「読者が正確に意図を受け取れるか」で測定される。美しい文章ではなく、正確で再現性のある情報伝達が目標である。

## ルールファイル構成

| ファイル | 対象 |
|---------|------|
| `preparation.md` | 執筆前の準備（アウトプット定義、読者設定、課題定義） |
| `document-structure.md` | 文書全体の構成とパラグラフ管理 |
| `clarity-and-precision.md` | 曖昧性の排除、事実と意見の分離、正確な表現 |
| `writing-style.md` | 文体の統一、文法的正確性、冗長性の排除 |
| `decision-documents.md` | 意思決定文書固有のルール（選択肢提示、提案構成） |
| `review-checklist.md` | 提出前の最終チェックリスト |

## 適用範囲

- AI-DLCの全フェーズ（INCEPTION / CONSTRUCTION / OPERATIONS）で生成される成果物
- 要件定義書、設計文書、レビュー文書、提案書、手順書
