# レビューチェックリスト

文書を提出・公開する前に、以下の項目を確認すること。

## 準備段階の確認

- [ ] アウトプット（読者が得る情報）が明確に定義されているか
- [ ] 解決すべき課題が正しく特定されているか
- [ ] 対象読者が明確で、その知識レベルに合った記述になっているか

## 構成の確認

- [ ] 冒頭に文書の目的が簡潔に記載されているか
- [ ] 結論が先に提示され、根拠がその後に続く構成か
- [ ] パラグラフごとにトピックセンテンスが冒頭にあるか
- [ ] 1パラグラフ1主題の原則が守られているか
- [ ] 15〜20分で読了できる分量か（意思決定文書の場合）

## 明確性の確認

- [ ] 曖昧な指示代名詞が残っていないか
- [ ] 同一語に複数の意味を持たせていないか
- [ ] 略語は初出時に完全形を記載しているか
- [ ] 事実と意見が明確に区別されているか
- [ ] 修飾関係が一意に特定できるか
- [ ] 主語と述語が論理的に対応しているか

## 文体・表記の確認

- [ ] 文体（です・ます / である）が統一されているか
- [ ] 技術用語の表記が統一されているか
- [ ] 不要な冗長表現が残っていないか
- [ ] 過度な修飾語がないか

## データ・参照の確認

- [ ] 数値が具体的に記載されているか（定性表現の排除）
- [ ] 図表に自然言語の説明が併記されているか
- [ ] 出典・リンクが正確で有効か
- [ ] コピー&ペースト由来の不整合が残っていないか

## 査読プロセスの確認

- [ ] Agent Teamsによるファクトチェックを実施したか
- [ ] 構成レビュー（結論先行、パラグラフ管理、トピックセンテンス）を実施したか
- [ ] 冗長性チェック（重複表現、過度な修飾語、不要な前置き）を実施したか

## 意思決定文書の追加確認

- [ ] 検討対象のすべての選択肢が提示されているか
- [ ] 比較項目が選択肢間で一貫しているか
- [ ] 著者としての第一候補と推薦理由が明示されているか
- [ ] One-way door / Two-way door の区別が記載されているか
- [ ] 棄却済みの選択肢とその理由が記載されているか
